# qrator_service (Resource)

Manages a service in Qrator — creates it via `service_create`, manages its name, status, and upstream IP addresses.

~> **Note:** Deleting this resource is a no-op — the service itself is not removed.

Use `qrator_service_services` to manage the service list, `qrator_service_sni` for SNI configuration, and `qrator_service_whitelist` / `qrator_service_blacklist` for IP lists.

## Example Usage

```terraform
resource "qrator_service" "example" {
  client_id = var.client_id
  name      = "my-service"
  ips       = ["10.0.0.1", "10.0.0.2"]
}
```

## Import

Existing services can be imported using the service ID:

```shell
terraform import qrator_service.example 12345
```

After import, the first `terraform plan` will show a diff for `client_id`, `name`, and `ips` — apply it to sync the state.
