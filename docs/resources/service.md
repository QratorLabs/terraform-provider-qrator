# qrator_service (Resource)

Manages a service in Qrator — its name and access policy.

~> **Note:** Deleting this resource is a no-op — the service itself is not removed.

Use `qrator_service_status` to manage the service status, `qrator_service_services` to manage the service list, `qrator_service_sni` for SNI configuration, and `qrator_service_whitelist` / `qrator_service_blacklist` for IP lists.

## Example Usage

```terraform
resource "qrator_service" "example" {
  id   = 12345
  name = "my-service"
}
```

### Restrict access to whitelisted IPs only

```terraform
resource "qrator_service" "restricted" {
  id                      = 12345
  name                    = "internal-service"
  not_whitelisted_policy  = "drop"
}

resource "qrator_service_whitelist" "allowed" {
  service_id = 12345

  entries = [
    {
      ip      = "203.0.113.10"
      comment = "Office gateway"
    }
  ]
}
```

## Import

Existing services can be imported using the service ID:

```shell
terraform import qrator_service.example 12345
```
