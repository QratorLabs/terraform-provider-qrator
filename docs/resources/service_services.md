# qrator_service_services (Resource)

Manages the service list for a service in Qrator. Supports DNS, HTTP, ICMP, NAT, any-ingress-egress, proto-ingress-egress, tcp-ingress-egress, tcp-egress, and frag-ingress-egress service types.

The API works on a bulk/replace basis: `services_set` accepts the entire list at once. There are no individual add/remove operations — the provider reads the current list, applies changes, and writes the full list back.

~> **Note:** Deleting this resource clears the service list (does not remove the service).

## Example Usage

### DNS service

```terraform
resource "qrator_service_services" "dns" {
  service_id = 12345

  dns = [
    {
      port = 53
    }
  ]
}
```

### HTTP with upstream SSL

```terraform
resource "qrator_service_services" "web" {
  service_id = 12345

  http = [
    {
      port         = 443
      ssl          = true
      http2        = true
      upstream_ssl = true
    }
  ]
}
```

~> **Validation:** `http2 = true` requires `ssl = true`. Setting `http2 = true` with `ssl = false` will produce a validation error.

### Multiple service types

```terraform
resource "qrator_service_services" "full" {
  service_id = 12345

  dns = [
    {
      port = 53
    }
  ]

  http = [
    {
      port         = 80
      upstream_ssl = false
    },
    {
      port         = 443
      ssl          = true
      http2        = true
      upstream_ssl = true
    }
  ]

  icmp = [
    {
      rate_limit = 80000
    }
  ]

  nat = [
    {
      port  = 53
      proto = "udp"
    },
    {
      port  = 25
      proto = "tcp"
    }
  ]

  any_ingress_egress = [
    {
      rate_limit = 80000
      drop_amp   = true
    }
  ]

  tcp_ingress_egress = [
    {}
  ]

  tcp_egress = [
    {}
  ]

  frag_ingress_egress = [
    {
      rate_limit = 80000
    }
  ]
}
```

## Import

Service configuration can be imported using the service ID:

```shell
terraform import qrator_service_services.example 12345
```

After import, run `terraform plan` to see the current state read from the API.

## Validation Rules

- No duplicate services with the same composite key (type + port/proto).
- At most one entry each for: `icmp`, `any_ingress_egress`, `tcp_ingress_egress`, `tcp_egress`, `frag_ingress_egress`.
- `drop_amp` and `rate_limit` are only valid for UDP services (nat).
- `http2 = true` requires `ssl = true`.
