# qrator_service_blacklist (Resource)

Manages the blacklist for a service in Qrator. The blacklist contains IP addresses that are always blocked from accessing the service.

By default, management is **exclusive**: any IP not present in `entries` is removed on `terraform apply`, including IPs added via the Qrator UI. Set `exclusive = false` to switch to additive mode, where externally added IPs are left untouched.

~> **Note:** Deleting this resource clears the entire blacklist.

## Example Usage

```terraform
resource "qrator_service_blacklist" "example" {
  service_id = 12345

  entries = {
    "203.0.113.1"   = { comment = "Known attacker" }
    "198.51.100.50" = { comment = "Temporary block (24h)", ttl = 86400 }
  }
}
```

### Additive mode — preserve IPs added outside Terraform

```terraform
resource "qrator_service_blacklist" "additive" {
  service_id = 12345
  exclusive   = false

  entries = {
    "203.0.113.1" = { comment = "Known attacker" }
  }
}
```

## Import

Blacklist can be imported using the service ID:

```shell
terraform import qrator_service_blacklist.example 12345
```

After import, `entries` in state will be empty. Run `terraform apply` to populate the blacklist from your configuration.
