# qrator_domain_whitelist (Resource)

Manages the whitelist for a domain in Qrator. The whitelist contains IP addresses that are always allowed to access the domain.

By default, management is **exclusive**: any IP not present in `entries` is removed on `terraform apply`, including IPs added via the Qrator UI. Set `exclusive = false` to switch to additive mode, where externally added IPs are left untouched.

~> **Note:** Deleting this resource clears the entire whitelist.

## Example Usage

### Basic

```terraform
resource "qrator_domain_whitelist" "example" {
  domain_id = 12345

  entries = {
    "203.0.113.10" = { comment = "Office" }
  }
}
```

### Restrict access to whitelisted IPs only

```terraform
resource "qrator_domain_whitelist" "restricted" {
  domain_id    = qrator_domain.example.id
  default_drop = true

  entries = {
    "203.0.113.10" = { comment = "Office gateway" }
  }
}
```

### Multiple IPs, one with TTL

```terraform
resource "qrator_domain_whitelist" "monitoring" {
  domain_id = 12345

  entries = {
    "198.51.100.1" = { comment = "Monitoring server" }
    "198.51.100.2" = { comment = "Temporary access", ttl = 3600 }
  }
}
```

### Additive mode — preserve IPs added outside Terraform

```terraform
resource "qrator_domain_whitelist" "additive" {
  domain_id = 12345
  exclusive  = false

  entries = {
    "203.0.113.10" = { comment = "Office gateway" }
  }
}
```

## Import

Whitelist can be imported using the domain ID:

```shell
terraform import qrator_domain_whitelist.example 12345
```

After import, `entries` in state will be empty. Run `terraform apply` to populate the whitelist from your configuration.
