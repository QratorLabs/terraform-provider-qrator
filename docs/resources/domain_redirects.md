# qrator_domain_redirects (Resource)

Manages HTTP redirect rules for a domain in Qrator.

Rules are evaluated in order. Matching is based on the incoming port, hostname, and URI path. A rule with a `null` redirect target disables matching and blocks less-specific (later) rules from firing — useful for carve-outs.

The API works on a bulk/replace basis: `redirect_set` writes the entire list at once. The provider preserves the order from the configuration.

~> **Note:** Deleting this resource clears all redirect rules (`redirect_set([])`). It does not remove the domain.

## Example Usage

### Redirect HTTP to HTTPS

```terraform
resource "qrator_domain_redirects" "tls" {
  domain_id = 12345

  redirects = [
    {
      from = {
        port     = 80
        hostname = { type = "any" }
        uri      = null  # match any URI
      }
      redirect = {
        code     = 301
        schema   = "https"
        hostname = null  # keep original hostname
        port     = 443
        path     = null  # keep original path
        args     = true  # keep query string
      }
    }
  ]
}
```

### Redirect a specific path, block others

Use a `null` redirect to prevent less-specific rules from matching certain paths.

```terraform
resource "qrator_domain_redirects" "mixed" {
  domain_id = 12345

  redirects = [
    # Block redirect for /api/* — the null target prevents fallthrough
    {
      from = {
        port     = 80
        hostname = { type = "any" }
        uri      = { type = "stop", value = "/api/" }
      }
      redirect = null
    },

    # Redirect everything else on port 80 to HTTPS
    {
      from = {
        port     = 80
        hostname = { type = "any" }
        uri      = null
      }
      redirect = {
        code     = 301
        schema   = "https"
        hostname = null
        port     = 443
        path     = null
        args     = true
      }
    }
  ]
}
```

### Permanent redirect for a renamed page

```terraform
resource "qrator_domain_redirects" "legacy" {
  domain_id = 12345

  redirects = [
    {
      from = {
        port     = 443
        hostname = { type = "fqdn", value = "example.com" }
        uri      = { type = "exact", value = "/old-page" }
      }
      redirect = {
        code     = 301
        hostname = null
        port     = 443
        path     = "/new-page"
        args     = false  # strip query string
      }
    }
  ]
}
```

## Import

The redirect configuration can be imported using the domain ID:

```shell
terraform import qrator_domain_redirects.example 12345
```

After import, run `terraform plan` — the state will be populated from the API.

## Validation Rules

- Maximum 70 redirect rules.
- `from.hostname.value` is required and must be non-empty when `from.hostname.type` is `"fqdn"`.
- Each `from` matcher must be unique (port + hostname type/value + URI type/value).
- Per port, at most one rule may have `from.hostname.type = "any"` (the API constraint *"only one 'any' per port"*).
- Per port, at most one rule may have `from.uri = null` (the API constraint *"only one 'default' per port"*).
