# qrator_domain (Resource)

Manages a domain in Qrator — creates it via `domain_create`, manages its name.

~> **Note:** Deleting this resource is a no-op — the domain itself is not removed.

Use `qrator_domain_services` to manage the service list, `qrator_domain_sni` for SNI configuration, and `qrator_domain_whitelist` / `qrator_domain_blacklist` for IP lists.

## Example Usage

```terraform
resource "qrator_domain" "example" {
  client_id = var.client_id
  name      = "example.com"
}
```

### With maintenance mode

```terraform
resource "qrator_domain" "example" {
  client_id         = var.client_id
  name              = "example.com"
  maintenance_until = 1775520000 # 2026-04-07T00:00:00Z
}
```

## Import

Existing domains can be imported using the domain ID:

```shell
terraform import qrator_domain.example 12345
```

After import, the first `terraform plan` will show a diff for `client_id` and `name` — apply it to sync the state.
